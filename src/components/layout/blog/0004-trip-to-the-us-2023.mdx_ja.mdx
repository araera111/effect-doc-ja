2023 年 10 月と 11 月、ヨハネス・シクリングとミレラ・プリフティと共に、米国を訪れて興味深い人々と出会い、サンフランシスコでミートアップを開催しました。

まず初めに、旅行仲間に感謝したいと思います。彼らのおかげで、この旅はより楽しく、興味深いものになり、ミートアップを最良の形で組織することができました。

ミートアップは、[Amplify Partners](https://www.amplifypartners.com/) の美しいオフィススペースで開催され、無料でホストしてもらいました。このような提案がなければ、同じクオリティを達成するのは非常に難しかったでしょう。本当に感謝しています。

ミートアップは多くの人に参加され、ショーン・ワン、セオ・ブラウン、ドミニク・トルノウ、デビッド・シーゲル、ティム・スチャネク、マリー・シュネガンズなど、非常に興味深い人々で埋め尽くされました。雰囲気はポジティブで、ピザを食べたりスパークリングウォーター（とビール）を飲んだりして、楽しい時間を過ごしました。

私たちは 3 つの講演を行いました：

- 開催の最初を飾ったのは私（マイケル・アルナルディ）で、「Effect の Why」について話しました。
- 続いてヨハネス・シクリングが、Effect を使用してオーバートーンを構築した経験について語りました。
- クローズを飾ったトーマス・バリンガーが、彼らの顧客が Effect を Convex と共にどのように利用しているかについて話しました。

ミートアップから多くを学び、セオから私の講演に対して貴重なフィードバックを直接受けました（彼には非常に感謝しています）。最も重要なこととして、録音機材を持参する必要性を学びましたので、[高品質の録音](https://www.youtube.com/playlist?list=PLDf3uQLaK2B_I-5aX9EtSE338qA06tWrB)がない理由がわかりました。

ミートアップ以外にも、多くの興味深い人々と出会い、彼らの時間と提供してくれた洞察に感謝しています。

会議からの主なポイントは以下の通りです：

- Effect の学習曲線はよりアクセスしやすくなっていますが、まだ少し急です。
- オブザーバビリティは誰もが望んでいるものですが、実現するのは難しいです。Effect はそこを支援できます。
- Effect が解決する問題は、実際に生産現場でリアルで大きなもので、経験したことがないと最初はそれに気づくのは難しいです。
- TypeScript はバックエンドシステムの開発において事実上の好ましい言語になりつつあり、フロントエンドだけでなく、Effect はその場所にぴったりです。
- 統合（それに伴うワークフロー）は大きな痛点であり、Effect の信頼性の側面が鍵になる可能性があります。

会議の記録は以下の通りです：

- マックスウェル・ブラウン：唯一無二の存在！NJ で素敵なディナーを共にし、Effect の未来について話し合いました。
- ケリー・マク：NY の[Work-Bench](https://www.work-bench.com/)の GP。素晴らしい会話を交わし、次回の訪問時にアジアンフードをテーマにした OSS ファウンダーディナーの開催に協力してくれることになりました。
- ジェイミー・ターナー：[Convex](https://www.convex.dev/)の CEO。開発ツールの構築とスケーリングに関するすべてについて話し合いました。
- トーマス・バリンガー：[Convex](https://www.convex.dev/)のシニアデベロッパー。彼らのユーザーがどのように Effect を活用しているか、そしてより深い統合から得られる利点について話し合いました。
- デビッド・クールシッド：[Stately](https://stately.ai/)の CEO であり、[XState](https://xstate.js.org/)の著者。ステートチャートとエフェクトシステムの相互の利点や、こうした複雑なシステムの学習曲線をどのように簡単に理解できるようにするかについて話し合いました。また、オープンソースファースト企業の構築に関する独自の洞察も伺いました。
- ジャレッド・サムナー：[Oven](https://oven.sh)の CEO であり、[Bun](https://bun.sh/)の著者。バックエンド言語としての JavaScript の重要性や、エコシステムの未来について議論しました。人々が好むツールを作るための洞察を多く得ることができました。
- ジェイク・クーパー：[Railway](https://railway.app/)の CEO。クラウド製品の構築とスケーリングに関連するあらゆるテーマについて議論し、Railway（または同様のもの）にデプロイするのに最も適したサービスとは何かについて多くの洞察を得ることができました。また、Effect Cluster のおかげで構築できるようになるワークフロー的なシステムがそのようなプラットフォームには最適であるという点について意見が一致しました。
- クリスチャン・グラッシオーニョ：[Seltzer.dev](Seltzer.dev)の創業者。データベースツールの構築に関する課題を学び、Effect がどのようにそれを支援できるかについて話し合いました。オープンソースでの構築に関するいくつかの洞察を共有し、SF で最も美味しくて高価と言われるサンドイッチを共にしました。
- ドミニク・トルノウ：[Resonate](https://www.resonatehq.io/)の CEO であり、[Temporal](https://temporal.io/)の元プリンシパル。耐久性コンピューティングのあらゆる側面について議論し、どのようにしてそのようなシステムを構築しスケールさせるか、また耐久性のある継続がほぼすべての言語で入手可能な将来はどのようになるかについて洞察を得ました。
- ジョシュ・カール：[Hightouch.io](https://hightouch.io/)の創業者兼 CTO。多くの異なるソースと連携する必要がある統合システムの構築に関する課題について話し合い、Effect がそれらの問題を解決する手助けができる方法について議論しました。
- タニシュク・カンチャーラ：フルスタックデベロッパー。Effect の学習に関する課題を話し合い、適切な命名規則を選択することの重要性について多くの洞察を得ました。
- ショーン・ワン：唯一無二の[Swyx](https://www.swyx.io/)。AI とワークフロー、テクノロジーを知らしめるための DevRel の側面に関する多くの洞察を得ました。スクラッチから会社を構築することについて話し合い、私たちがすべて直面する課題について意見を交わしました。
- ギレルモ・ラウチ：[Vercel](https://vercel.com/)の CEO であり、Next.js の著者。ギレルモとの会話の深さを数語で表現するのは難しいですが、Effect と Next.js を介した RSC のプロトタイプ統合を示した際に「全く理解できる」と反応してくれたのは驚きでした。同じビジョンを共有し、フロントエンドとバックエンドの境界を曖昧にすることの重要性、スケールでのオブザーバビリティとテスタビリティの重要性について深く一致しました。冗談のように（でもあまり笑えない）Effect の上に Next.js と RSC が将来的に構築されなければならないという話も出ました（現実的であるかどうかは別として）。
- チェン・ルー：[Midjourney](https://www.midjourney.com/home)のソフトウェアエンジニア。チェンはエフェクトシステムと言語設計の先駆者です。エフェクトシステムのあらゆる側面、課題、潜在的な欠点について議論しました。チェンが取り組んでいることの一部を知ることができ、まさに驚愕でした。フルセルフドライブを目撃するという素晴らしい経験もしました。
- ブライアン・ダグラス：[Open Sauced](https://opensauced.pizza/)の CEO であり、元 Github の開発者支援ディレクター。オープンソースと開発者支援について議論し、成功したオープンソースプロジェクトを運営する際の重要な洞察を多数学びました。
- セオ・ブラウンとマーク・R・フロルコフスキー：[Ping.gg](https://ping.gg)の CEO および CTO。大衆向けツールを作る際の重要なポイントや、その人気を得る方法について多くのことを学びました。現在彼らが UploadThing に直面している課題について議論し、Effect がどのようにそれを助けられるかを話し合いました。ヨハネスは、彼らのコードベースで Effect を使用するためのハッキングセッションを始めました。
- トム・プレストン・ウェルナー：[Github](https://github.com/)の創業者で元 CEO、[Redwood](https://redwoodjs.com/)の著者、オープンソースの先駆者。オープンソースとオープンソースファースト企業、フレームワーク、開発の未来について話し合い、素晴らしいランチを共にし、環境保護の重要性についても議論しました。
- ジェイソン・ラスター：[Replay.io](https://replay.io)の CEO。Jason と他のデベロッパーツールの創業者との素敵な「DevTools」ディナーを楽しみ、デバッグツールの構築に関する課題について学び、デバッグの未来を垣間見ることができました。
